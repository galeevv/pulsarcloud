# Pulsar 2.0 audit and deployment status — 2026-07-13

## Verdict

The project is deployable in a safe limited production mode, but it does not
yet satisfy every item in `START_PROMT.md`. The public site, backend, durable
worker, SQLite database, Remnawave Panel, and Remnawave subscription page are
deployed. Billing remains deliberately disabled because the application's
The Remnawave 2.8.0 HTTP provider is implemented and its management-plane contract is covered by unit tests plus a live create/update/read/URL-rotation/delete smoke test. A real traffic Node/Host and the complete paid acceptance flow are still absent, so this does not yet deliver usable VPN connectivity.

## Verified and fixed before deployment

- Next.js 16.2.6 implementation was checked against the local versioned guides.
- TypeScript, ESLint, all automated tests, and production builds pass on Windows
  and on the Linux VPS.
- Telegram webhook storage no longer persists raw `/start` bearer tokens.
- Telegram login completion is bound to the browser that initiated the flow.
- A real production runtime cannot silently fall back to development/test mode.
- Readiness requires every expected migration to be complete.
- Expired `PENDING` payments no longer block a replacement checkout.
- Unchanged Remnawave reconciliation advances `lastSyncedAt` and cannot starve
  later subscriptions.
- Heavy GIF assets are no longer eagerly preloaded by every card.
- Support chat refresh/error handling, page metadata, headings, and bottom-nav
  accessibility were improved using the installed shadcn composition.
- Anti-framing headers are emitted by both Next.js and Nginx.

## Live deployment

- `https://pulsar-cloud.space` — Pulsar web/backend.
- `https://panel.pulsar-cloud.space` — Remnawave Panel 2.8.0.
- `https://sub.pulsar-cloud.space/<shortUuid>` — subscription page 7.2.6.
- Host Nginx is the only public listener on ports 80/443. Pulsar and all
  Remnawave services listen on loopback-only ports.
- One SAN Let's Encrypt certificate covers the three hostnames; renewal dry-run
  succeeds.
- UFW allows only 22/80/443, SSH passwords are disabled, and fail2ban is active.
- SQLite WAL database, six-hour verified backups, and the web/worker services
  run under an unprivileged `pulsar` account.
- Telegram webhook is installed with its secret header and reports no pending
  update/error.
- `BILLING_ENABLED=false` is intentional.

The initial Remnawave admin credentials are not stored in the repository or
deployment logs. Retrieve them on the VPS with:

```bash
sudo cat /root/remnawave-initial-admin.txt
```

Delete that file after saving the credentials in a password manager.
