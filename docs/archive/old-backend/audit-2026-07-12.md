# Repository and VPS audit — 2026-07-12

## Summary

The database foundation, billing confirmation transaction, SQLite runtime, and
outbox boundaries are strong. Baseline typecheck, lint, tests, and production
build passed before changes. The main gaps were incomplete referral onboarding,
manual-only test payments, missing identity linking, admin mutations bypassing
outbox synchronization, misleading turnover, and an unconfigured Remnawave Node.

## Findings

### Critical — resolved

- Remnawave Panel had profiles and squads but no Node; issued subscriptions
  could not carry working endpoints.
- A Prisma table rebuild would have silently removed manual CHECK constraints
  and billing immutability triggers. The migration now explicitly recreates
  them.

### High — resolved

- Referral registrations did not receive the required three-day trial.
- Telegram referral registration discarded the invite code.
- Admin extension changed local expiry without scheduling Remnawave sync.
- Test/mock ledger movements polluted the displayed live turnover.
- Users could not link a second authentication identity.

### Medium — resolved

- Test payment required a manual admin action and could not exercise the normal
  user checkout flow.
- Admin pricing omitted device bounds and duration discounts.
- Admin subscription sync was labelled and implemented as mock behavior.
- Remnawave standard/LTE squads were incomplete and had no active Node.

### Remaining high risk

- Immediate 75 RUB registration rewards are farmable. Before public payouts,
  introduce a hold, antifraud checks, or first-live-payment qualification.
- Platega does not document creation idempotency. A timeout after remote creation
  can leave an orphan provider transaction; add scheduled reconciliation.
- Previously shared Telegram, Resend, Platega, and Remnawave credentials must be
  rotated before accepting real users.

### Remaining medium risk

- The same VPS is a single point of failure for Panel, test Node, Pulsar, worker,
  and SQLite.
- Telegram sends do not provide provider-side idempotency; a narrow crash window
  can duplicate a notification.
- HWID adapter methods exist, but the subscription UI still renders placeholder
  device slots instead of live device data.
- Local source output can appear as mojibake in legacy Windows PowerShell because
  UTF-8 files have no BOM; Node, Next.js, and browsers read them as UTF-8.

## Keep

- Prisma domain model and immutable quote/payment/period boundaries.
- `runInTransaction` busy retry and SQLite startup checks.
- payment provider interface and mock provider.
- outbox worker and idempotency constraints.
- current user-facing layout and component system.

## Continue improving

- Split the large admin action module into domain-specific action files.
- Add pagination to large admin tables.
- Add payment reconciliation and job lock recovery.
- Add live HWID device management to `/subscription`.
- Add integration contract tests against provider sandboxes.
